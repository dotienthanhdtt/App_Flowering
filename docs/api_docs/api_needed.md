## A. Backend — Missing Endpoints (Not Yet Built)

### Forgot Password Flow

#### POST /auth/forgot-password

Send a 6-digit OTP to the user's email.

**Auth:** Public

**Request Body**
```json
{
  "email": "user@example.com"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `email` | string | yes | Registered email address |

**Response 200**
```json
{
  "code": 1,
  "message": "Verification code sent",
  "data": {
    "email": "u***@example.com"
  }
}
```

**Errors**
- `404` — Email not found
- `429` — Too many requests (rate limit: 3/hour)

---

#### POST /auth/verify-otp

Verify the 6-digit OTP received via email.

**Auth:** Public

**Request Body**
```json
{
  "email": "user@example.com",
  "otp": "123456"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `email` | string | yes | Email that received the OTP |
| `otp` | string | yes | 6-digit code (valid 10 minutes) |

**Response 200**
```json
{
  "code": 1,
  "message": "OTP verified",
  "data": {
    "resetToken": "uuid-short-lived-token"
  }
}
```

| Field | Description |
|---|---|
| `resetToken` | One-time token used to call `/auth/reset-password`. Valid for 15 minutes. |

**Errors**
- `400` — Invalid or expired OTP
- `429` — Too many attempts (max 5)

---

#### POST /auth/reset-password

Set a new password using the verified reset token.

**Auth:** Public

**Request Body**
```json
{
  "resetToken": "uuid-short-lived-token",
  "newPassword": "NewSecurePass123!"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `resetToken` | string | yes | Token from `/auth/verify-otp` |
| `newPassword` | string | yes | Min 8 characters |

**Response 200**
```json
{
  "code": 1,
  "message": "Password reset successfully",
  "data": null
}
```

**Errors**
- `400` — Weak password or invalid/expired reset token
- `401` — Token already used

## B. Backend — Response Extensions Needed

### POST /onboarding/complete — Add `scenarios` Array

Current response only returns a profile object. The Scenario Gift screen (Screen 08) requires 5 AI-generated personalized scenario cards.

**Updated Response 200**
```json
{
  "code": 1,
  "message": "Success",
  "data": {
    "name": "Thanh",
    "nativeLanguage": "vi",
    "targetLanguage": "en",
    "currentLevel": "B1",
    "learningGoals": ["business communication", "travel"],
    "weeklyAvailability": "5 hours",
    "preferredTopics": ["technology", "sports"],
    "scenarios": [
      {
        "id": "uuid",
        "title": "Job Interview at a Tech Company",
        "description": "Practice answering interview questions for a role at a startup.",
        "icon": "briefcase",
        "accentColor": "primary"
      },
      {
        "id": "uuid",
        "title": "Ordering Coffee at a Café",
        "description": "Learn to order drinks, ask about the menu, and make small talk.",
        "icon": "coffee",
        "accentColor": "blue"
      }
    ]
  }
}
```

**`scenarios` item fields:**

| Field | Type | Description |
|---|---|---|
| `id` | UUID | Scenario identifier |
| `title` | string | Short scenario title |
| `description` | string | 1-2 sentence description |
| `icon` | string | Lucide icon name |
| `accentColor` | `primary \| blue \| green \| lavender \| rose` | Card accent color token |

**Constraint:** Always returns exactly 5 scenarios. Generated by AI based on `learningGoals` and `preferredTopics`.
